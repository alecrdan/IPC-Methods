This project compares the performance of transferring a large file between two processes using either a TCP port or a Unix Domain Socket (UDS).

## What It Runs

- TCP server/client: `port/process_a.py` (server) and `port/process_b.py` (client).
- UDS server/client: `unix_socket/process_a.py` (server) and `unix_socket/process_b.py` (client).
- Shared helpers: `lib/ipc_lib.py` for socket setup and file transfer.
- Test data: `10gb_file.txt` generated by `utils/generate_10gb_txt.py`.

## How to Run

From the `SpeedTest/` directory:

```sh
./run.sh
```

`run.sh` will:
1. Generate `10gb_file.txt` if it does not exist.
2. Start the TCP server, time the client transfer, then stop the server.
3. Start the UDS server, time the client transfer, then stop the server.

Notes:
- The default file size is 10 GiB. You can lower it by editing `DEFAULT_SIZE_BYTES` in `utils/generate_10gb_txt.py`.
- The UDS path defaults to `/tmp/speedtest.sock`.
- The script uses `/usr/bin/time -p` for timing and `pkill` to clean up prior runs.

## Manual Run (Optional)

TCP:
```sh
python3 port/process_a.py
python3 port/process_b.py
```

UDS:
```sh
python3 unix_socket/process_a.py
RUN_ONCE=1 python3 unix_socket/process_b.py
```
